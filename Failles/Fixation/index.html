Fixation de session

La faille de fixattion de session peut se produire si le site n'est pas fiable et comporte des vulnérabilitées tel que nous avons vu durant ce projet.

Comment réaliser la faille :

  • 1) Pour ce faire L'attaquant crée une session sur le serveur;
  • 2) L'attaquant transmet ce numéro de session à la victime, par exemple via un lien;
  • 3) La victime s'authentifie, en utilisant ce numéro de session;
  • 4) L'attaquant peut maintenant utiliser le compte de la victime.

Comment fonctionne la demo ?

Après avoir cliquer sur un bouton vous arriverez sur la page
attaquant.php
qui représente la page de l'attaquand. Imaginons qu'il mette la main sur l'id de session avec session_id il va créer le lien suivant : .../.../victime.php?id=123456
et l'envoyer à sa victime une fois que la victime à cliqué sur le lien une session va se créer pour lui avec comme identifiant ce qui se trouve après l'id -> donc le même id de session que l'attaquant. Donc tout ce que le site enregistrera pour la victime sera aussi enregistré pour l'attaquant. Pour utiliser la démo il faut :

  • 1) Cliquer sur simuler l'ouverture du lien;
  • 2) Garder l'onglet attaquant.php OUVERT de côté;
  • 3) Rentrer votre information super secrete dans l'onglet victime.php;
  • 4) Une fois celle-ci validé vous aurez juste à retourner sur attaquant.php et actualiser la page pour voir les informations rentrées par la victime
Dans la version non sécurisé ?

On garde le même identifiant de session tout au long de la navigation lors du site donc la victime et l'attaquant auront toujours la même id de session, donc ils seront à jamais synchroniser l'un à l'autre

Dans la version sécurisé ?

Pour chaque action on utilise :
session_regenerate_id()
afin que l'id de la session soit généré aléatoirement meme si on force l'id de la session. Ce qui à pour conséquence de désynchorniser les sessions de l'attaquant et de la victime
