Injection SQL

La faille SQLi, abréviation de SQL Injection, soit injection SQL en français, est un groupe de méthodes d'exploitation de faille de sécurité d'une application interagissant avec une base de données. Elle permet d'injecter dans la requête SQL en cours un morceau de requête non prévu par le système et pouvant en compromettre la sécurité.
Source : Wikipedia

Comment fonctionne la demo ?

Dans les deux versions la liste des identifiants et affichés afin de voir les utilisteurs disponoble. Vous trouverez deux champs afin de vous "connecter" , voici des idententifiants pour tester :
identifiant : root
mot de passe : 1234
après avoir tester la concordance avec les logs dans la base de données les informations de cet utilisteur seront affichés Pour tester une injection je vous invite à essayer : :
identifiant : root
mot de passe : 'OR 1=1 OR 1='

Dans la version non sécurisé ?

On intéroge la base tel quel sans traitement sur l'élément rentré ce qui permet toutes sortes d'injection.

Dans la version sécurisé ?

Pour chaque information rentrées on utilise :
addslashes("texte à traiter")
avant l'envoie à la base de données ce qui évitera les injections
